   Desenvolvido em 1991 por Tim Berners-Lee, o HTML (Linguagem de Marcação de Hipertexto)
foi uma linguagem que surgiu para ser entendida por meios de acesso diferentes.
O HTML ganhou força com a popularização do browser "Mosaic",
e foi usado para publicação de textos, imagens, vídeos, áudios e outros na web.
Ele é baseado no hipertexto, que são elementos como imagens, documentos, palavras etc.
que formam uma rede de informação quando conectados,
essa conexão permite a comunicação de dados, organizando conhecimentos e guardando informações.
Entre 1993 e 1995 surgiram novas versões do HTML,
que propunham mudanças para enriquecer as possibilidades de linguagens,
mas ainda não tratadas como padrão.
Em 1997, o grupo de trabalho W3C trabalhou na versão 3.2,
a qual foi tratada como prática comum.
Em 2004 foi fundado o WHATWG (Web Hypertext Application Technology Working Group)
por desenvolvedores como Mozilla, Apple e Opera,
o qual começou a escrever a nova versão do HTML, o HTML 5.
O HTML 5 é a nova versão do HTML 4, e facilita a manipulação de elementos,
permitindo o desenvolvedor modificar
de forma não intrusiva as características dos objetos. Diferentemente de versões anteriores, o HTML 5
tem ferramentas que permitem que o CSS e o JavaScript
façam seu trabalho da melhor forma ainda mantendo o site ou aplicativo leve e funcional.
Mudanças:
 -Estrutura:
   -O DOCTYPE sofreu uma pequena alteração, agora não é mais necessário referenciar o DTD, essa tarefa fica para o próprio navegador.
 -Tags modificadas:
   -Algumas tags sofreram alterações para serem mais semânticas.
 -Tags novas:
   -Tags surgiram para auxiliar na questão da interpolaridade e para substituir plug-ins e tags antigas.
- Tags descontinuadas:
   -Algumas tags não serão mais utilizadas por serem substituídas.
 -Interpolável:
   -O HTML 5 é interpolável, ou seja, foi desenvolvido pensando em dispositivos futuros, possibilitando a reutilização de informações.
 -Mudanças sem necessitar de plug-ins:
   -Antes algumas coisas eram possíveis apenas com plug-ins, o que podia ocasionar em perda de performance, mas agora tags os substituiram.
 -Mais semântica:
   -O HTML 5 permite que o texto seja mais semântico com menos códigos, o que facilita para o browser entender as tags.

O que é JavaScript?


   Na década de 90 uma grande disputa no mercado de navegdores estava ocorrendo,
a Netscape buscava s destacar nessa "guerra" e planejava construir algo
para deixar a navegação mais dinâmica, o que seria um grande diferencial
na época, já que era preciso muito tempo para carregar uma página simples.
Em 1995, a Netscape contrata o desenvolvedor Brendan Eich para criar uma
linguagem de script para trazer "vida" às páginas da internet.
Assim foi criada uma linguagem chamada de LiveScript. Após uma parceria
com a Sun Microsystems, responsável pela crição da linguagem Java,
o que fez o nome LiveScript mudar para JavaScript, mudança a qual
foi uma jogada de markting com o objetivo de chamar atenção para o JavaScript,
já que o Java tinha conquistado uma boa fama.
A única semelhança entre Java e JavaScript é o nome, pois são linguagens completamente diferentes.
Em 1997, em associação com o ECMA, o JavaScript foi padronizado
para garantir o crescimento da linguagem, chamado de ECMAScript.
   O JavaScript ou JS é uma linguagem de programação de alto nível
e a mais popular no mundo de acoro com a Pesquisa de Desenvolvedores do Stack Overflow 2022,
devido ao JavaScript ser a linguagem padrão interpretada por navegadores
e formar a base da web com o HTML e o CSS. O JavaScript possui várias vantagens
por ter uma comunidade forte e consolidada, além de uma enorme evolução, como:
  ♦ Atualizações frequentes
  ♦ Correções de bugs
  ♦ Segurança
  ♦ Ajuda de outros desenvolvedores
  ♦ Criação de novas bibliotecas e códigos prontos
   O JavaSript ser uma linguagem de alto nível significa que ele torna o programa
muito mais compreensível para os humanos e precisa de menos linhas de código.

O que é JQUERY?



jQuery é uma biblioteca popular do JavaScript. Ela foi criado por John Resig em 2006
com o propósito de facilitar a vida dos desenvolvedores que usam JavaScript nos
seus sites. Não é uma linguagem de programação separada, funciona em conjunto
com o JavaScript. Com jQuery, você vai conseguir fazer muito mais com menos.

Criar códigos pode ser um pouco cansativo, especialmente quando isso envolve
muitas strings. A jQuery compacta várias linhas de código em uma única função.
Assim, você não precisa reescrever todos os blocos repetidamente para concluir sua tarefa.
